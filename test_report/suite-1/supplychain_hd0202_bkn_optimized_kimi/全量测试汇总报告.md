### 数据缺口处理策略

1. **基准数据存在，知识网络不存在**：基于基准数据回答，说明知识网络中未找到
2. **基准数据未提供完整信息**：说明数据缺口，提供查询路径建议
3. **知识网络存在**：使用知识网络数据验证和补充

## 6. 测试方法说明

### 遵循的规则

1. **串行回答**：按 Q01→Q50 顺序逐一回答，不并行
2. **禁止虚构数据**：对数据缺口明确说明，不编造供应商、单号、日期
3. **结论优先**：每题回答首句即明确结论（是/否/高风险/低风险）
4. **行动导向**：包含可执行动作（谁、做什么、今天/本周）
5. **合理追问**：结尾有与问题相关的追问引导

### 评分标准遵循


| 不合格模式  | 本次测试表现          |
| ------ | --------------- |
| 数据列表开场 | 避免，首句即结论        |
| 风险低估   | 准确识别风险等级        |
| 虚构数据   | 无虚构，数据缺口明确说明    |
| 无行动    | 每题都有行动建议        |
| 仓库不分   | 明确区分有效仓/委外仓/借用仓 |
| MRP 误读 | 准确区分 MRP 投放状态   |

## 7. 主要发现

### 业务洞察

1. **T01-000080 风险可控但紧张**：PCB 是关键瓶颈，需要重点关注
2. **948-000077 风险极高**：MRP 未投放、PR/PO 为零，基本确定延期
3. **两个产品对比**：T01 执行面领先 4-6 周，948 需要专项追赶
4. **物料复杂性差异**：T01 22 种物料，948 601 种物料，复杂度差异大

### 系统能力验证

1. **知识网络查询**：通过 bkn 接口成功查询对象类型、关系类型
2. **上下文检索**：通过 context-loader 获取业务规则
3. **数据关联**：理解 Forecast→MRP→PR→PO 的溯源链路
4. **业务规则应用**：正确应用齐套判断、风险预警等业务规则
