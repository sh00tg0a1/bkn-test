## 关键发现

### 1. 数据源完全一致
数据库中的数据与 BKN 优化版测试中获取的数据**完全一致**，基准数据 11/11 全部匹配。说明 BKN 的数据视图正确映射了底层数据库。

### 2. 新增可用信息
通过数据库直连获取了以下 BKN 测试中难以获取的信息：

| 信息类型 | 详情 |
|---------|------|
| PCB 历史供应商 | 深圳市浩达多层线路板有限公司（3次采购历史） |
| 安全芯片供应商 | 北京宏思电子技术有限责任公司 |
| 连接器供应商 | 深圳市扬展康电子有限公司 |
| PCBA 委外供应商 | 乾创电子（苏州）有限公司 |
| PO 精确交货日 | 大部分 04-28，最早 03-24（128-000629 已超期） |
| 已到货数据 | 包装箱(60)、衬板(240)、贴纸(3000) 已到货 |

### 3. 超期预警
- **128-000629（屏蔽罩B上盖）**: 交货日 2026-03-24，截至测试日(03-22)仅剩 2 天，invqty=0（尚未到货）
- 该信息在 BKN 测试中因为缺少 deliverdate 字段的明确显示而不易发现

### 4. 历史数据陷阱验证
- PCB(134-000396) 在 PR 表中有 4 条记录，但 `srcbillnumber` 为空字符串，说明这些 PR 不是从 MRP 投放的
- 直接用 `material_number = '134-000396'` 查 PR 会返回 4 条结果，造成"有 PR"的误判
- 必须通过 MRP `billno` → PR `srcbillnumber` 的链路关联才能正确判断当前批次的 PR 覆盖情况
- **这验证了 SKILL.md 中"当前批次数据查询"规则的必要性**
