## 三、Skill 优化建议

### 3.1 条件操作符问题（影响：错误率、重试成本）

| 问题 | 现状 | 建议 |
|------|------|------|
| `eq` 操作符不支持 | Skill 文档 `json-formats.md` 列出 `eq` 为有效操作符，但平台实际返回 400 | **Skill 应明确标注**：`object-type query` 仅支持 `like` / `in` / `not_like` / `not_in` / `exist` / `not_exist`；`eq` / `neq` / `gt` / `gte` / `lt` / `lte` / `match` 均不可用 |
| `match` 导致 500 | 服务端未优雅降级 | Skill 应在 `json-formats.md` 操作符表中标注 `match` 不支持 SQL 视图数据源 |
| 空条件查询失败 | `{"operation":"and","sub_conditions":[]}` 返回 400 | Skill 应注明空条件查询需使用 `--limit N` 参数形式，不能传空 condition body |

**预计收益**：消除 Q01 的 2 次错误调用 + AI 首次使用时的 trial-and-error，节省约 8s/轮。

### 3.2 分页效率问题（影响：调用次数、延迟）

| 问题 | 现状 | 建议 |
|------|------|------|
| 大结果集分页 | Q06 查 601 条 MRP 需 13 次调用（每页 50），占总调用 25% | **建议 1**: Skill 提示可用 `--limit 200` 或更大值减少分页次数 |
| 仅需统计时仍全量拉取 | 有时只需计数（如"总共多少条 MRP"），却要翻全部页 | **建议 2**: 提供计数查询模式或在 Skill 中说明如何高效获取总数 |

**预计收益**：Q06 从 13 次降至 3-4 次调用，节省约 40s。

### 3.3 历史数据过滤问题（影响：答案准确性）

| 问题 | 现状 | 建议 |
|------|------|------|
| PR/PO 返回全量历史 | 查 134-000396 的 PR 返回 4 条全是历史单据 | **Skill 应强调** `biztime >= demandStart` 过滤规则（BKN `forecast.bkn` 和 `network.bkn` 已写明，但 Skill 查询示例未体现） |
| MPS 返回全量历史 | 查 T01-000080 工单返回 12 条含大量历史完工单 | **Skill 应提供** `sourcebillnumber` 关联自制件 MRP 的查询模板 |

**预计收益**：避免 AI 误将历史 PR/PO 当作当前有效采购记录，防止"数据列表开场"和"风险低估"两种不合格回答模式。

### 3.4 查询模板建议

Skill 中应增加以下常用查询模板：

```bash
# 1. 查指定预测单的 MRP（推荐：首选查询）
kweaver bkn object-type query <kn-id> supplychain_hd0202_mrp \
  '{"limit":50,"condition":{"operation":"and","sub_conditions":[
    {"field":"rootdemandbillno","operation":"like","value":"<billno>"}
  ]}}'

# 2. 查指定物料的 MRP（在特定预测单下）
kweaver bkn object-type query <kn-id> supplychain_hd0202_mrp \
  '{"limit":10,"condition":{"operation":"and","sub_conditions":[
    {"field":"rootdemandbillno","operation":"like","value":"<billno>"},
    {"field":"materialplanid_number","operation":"like","value":"<material_code>"}
  ]}}'

# 3. 查有效仓库的库存（查完后按 warehouse 字段过滤有效仓库）
kweaver bkn object-type query <kn-id> supplychain_hd0202_inventory \
  '{"limit":50,"condition":{"operation":"and","sub_conditions":[
    {"field":"material_code","operation":"like","value":"<material_code>"}
  ]}}'

# 4. 查指定来源的生产工单（按自制件 MRP billno 关联）
kweaver bkn object-type query <kn-id> supplychain_hd0202_mps \
  '{"limit":20,"condition":{"operation":"and","sub_conditions":[
    {"field":"sourcebillnumber","operation":"like","value":"JHJY-20260306"}
  ]}}'
```

### 3.5 优先级排序

| 优先级 | 优化项 | 理由 |
|--------|-------|------|
| P0 | 修正操作符文档 | 每次新对话都会踩坑，直接影响首次调用成功率 |
| P1 | 增加 PR/PO 时间过滤说明 | 影响答案准确性（核心评分维度） |
| P1 | 增加查询模板 | 降低 AI 构造查询的复杂度 |
| P2 | 分页效率优化 | 影响响应速度但不影响正确性 |
| P2 | MPS 溯源查询说明 | 避免历史工单干扰 |
