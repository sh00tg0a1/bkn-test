# 动态计划协同 · 评分标准与测试方法

> 版本: v1.0

---

## 一、评分维度（每题）

| 维度 | 满分 | 评分标准 |
|------|------|---------|
| 准确性 | 40分 | 关键数字（数量、日期、单号、状态）与基准一致，不得虚构 |
| 结论优先 | 20分 | 回答首句即结论，明确是/否/高风险/低风险 |
| 行动导向 | 25分 | 包含可执行动作（谁、做什么、今天/本周） |
| 连贯性 | 15分 | 结尾有合理追问引导，方向匹配当前问题 |

### 合格门槛

- 单题总分 >= 70 分为通过
- 数据事实差异一票否决：关键数据（数量/日期/单号/状态）与金标答案不符，该题总分强制为 0
- 虚构数字（供应商/单号/到货日等）该题总分强制为 0

### 快速排查不合格模式

| 不合格模式 | 常见表现 |
|-----------|---------|
| 数据列表开场 | 回答前 3 行全是数据，没有结论 |
| 风险低估 | 把“PO正常”等同“无风险” |
| 虚构数据 | 捏造供应商名、单号、到货日期 |
| 无行动 | 只有描述，没有执行建议 |
| 仓库不分 | 把委外仓/借用仓当有效生产库存 |
| MRP误读 | 把“MRP未投放”误读为“已有采购动作” |

## 二、测试方法（强制流程）

0. **每次测试创建新目录**，目录内至少包含：
   - `result/`（放测试结果）
   - 报告文件（汇总报告 + 分场景报告）
1. **务必通过 KWeaver-Core SKILL 进行测试**（禁止绕过该工具链）
2. **按顺序加载并测试 S1 -> S2 -> ... -> S7 问题**，逐题尝试回答
3. **测试完成后，将答案输出到 `result/` 目录**
4. **根据答案进行 `result` 评估**（按本文件评分维度）
5. **输出最终报告**，必须包含：
   - 调用成功率
   - 调用次数
   - 回答准确率

## 四、准确性评价方法（Agent 回答 vs 金标答案）

### 4.1 评价原则

| 评价维度 | 说明 |
|---------|-----|
| **数据对比** | Agent 回答中的关键数字与金标答案对比 |
| **一致性判定** | ✅ 一致 / ⚠️ 部分一致 / ❌ 不一致 |
| **准确性评分** | 40 分制，存在数据事实差异直接否决 |
| **差异说明** | 记录差异原因（数据缺口/主观判断/计算错误等） |

### 4.2 对比表格模板

每道题应包含以下对比维度：

| 对比项 | 金标答案 | Agent 回答 | 是否一致 | 准确性得分 |
|-------|---------|-----------|---------|-----------|
| **结论** | 金标结论要点 | Agent 实际结论 | ✅/⚠️/❌ | - |
| **关键数据** | 金标关键数字 | Agent 回答数字 | ✅/⚠️/❌ | - |
| **行动建议** | 金标行动要点 | Agent 行动建议 | ✅/⚠️/❌ | - |
| **数据缺口处理** | - | 是否明确说明数据缺口 | ✅/⚠️/❌ | - |
| **虚构数据** | - | 是否虚构供应商/单号/日期 | ✅/❌ | - |

### 4.3 准确性评分细则

| 评分项 | 分值 | 评分标准 |
|-------|-----|---------|
| 关键数字准确 | 20 分 | 数量/日期/单号/状态与金标一致 |
| 无虚构数据 | 10 分 | 未捏造供应商名、单号、到货日期等 |
| 数据缺口说明 | 5 分 | 对无法确认的数据明确说明缺口 |
| 计算逻辑正确 | 5 分 | 计算过程（如齐套率、完成率）逻辑正确 |

### 4.4 一票否决触发条件

| 触发条件 | 处理方式 |
|---------|---------|
| 关键数据与金标答案存在事实差异（数量/日期/状态不符） | 该题总分强制为 0 分 |
| 虚构关键数据（供应商/单号/到货日） | 该题总分强制为 0 分 |
| 结论完全错误（如风险判断相反） | 该题总分强制为 0 分 |

### 4.5 常见差异类型及扣分标准

| 差异类型 | 扣分 | 说明 |
|---------|-----|-----|
| 数据列表开场（前3行无结论） | -5 分 | 结论优先维度 |
| 关键数字错误 | -5~10 分 | 准确性维度 |
| 对数据缺口物料未提及 | -3~5 分 | 准确性维度 |
| 未通过知识网络验证 | -4 分 | 准确性维度（产品不在 KN 中） |
| 主观判断无数据支撑 | -3~5 分 | 准确性维度 |
| 未说明计算过程 | -2 分 | 准确性维度 |
| 行动建议不完整 | -3 分 | 行动导向维度 |
| 追问引导不合理 | -1~2 分 | 连贯性维度 |

## 五、测试报告必须包含的内容

### 5.1 报告文件清单

| 文件 | 说明 |
|-----|-----|
| `全量测试汇总报告.md` | 整体统计、核心结果、准确性分析 |
| `答案对比汇总报告.md` | 50 题金标答案 vs Agent 回答逐题对比 |
| `S1_交期判断_测试报告.md` | Q01-Q08 详细回答 + 准确性分析 |
| `S2_晨会全局扫描_测试报告.md` | Q09-Q14 详细回答 + 准确性分析 |
| `S3_物料跟催_测试报告.md` | Q15-Q22 详细回答 + 准确性分析 |
| `S4_风险预警播报_测试报告.md` | Q23-Q29 详细回答 + 准确性分析 |
| `S5_计划执行日报_测试报告.md` | Q30-Q35 详细回答 + 准确性分析 |
| `S6_深度追问与综合场景_测试报告.md` | Q36-Q45 详细回答 + 准确性分析 |
| `S7_边界与异常处理_测试报告.md` | Q46-Q50 详细回答 + 准确性分析 |

### 5.2 每题必须包含的对比要素

- Agent 回答（实际回答内容）
- 金标答案（来自 test-cases/answers/*.md 的参考要点）
- 对比分析表（结论/关键数据/行动建议/连贯性 四维度评分）
- 差异说明（详细说明差异原因、扣分依据）
- 准确性评价（关键数字准确性、数据缺口处理、虚构数据、计算逻辑）

### 5.3 汇总报告必须包含的统计

| 统计项 | 说明 |
|-------|-----|
| 总题量 | 50 题 |
| 通过题数 | 总分 >=70 且准确性 >=25 |
| 被否决题数 | 准确性 <25 分（一票否决） |
| 通过率 | 通过题数 / 总题数 |
| 平均总分 | 所有题目平均分 |
| 平均准确性 | 准确性维度平均分 |
| 准确性分布 | 各分数段题目数量统计 |
| 与金标答案一致性 | 高度一致 / 部分一致 / 数据缺口 题目分布 |
