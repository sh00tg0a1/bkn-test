# Suite-3 · PMC 智能问答系统测试集 · 评分标准与测试方法

> 版本: v1.0（Suite-3）  
> 数据基线：`ref/suite-3/supplychaindata_1/`

---

## 一、评分原则（总则）

与 Suite-2 一致：**以答案是否正确为准**。

- **正确**：被测回答中**所有数值**与**所有要素**均与金标标准答案一致（按本题所属行为类型的附加规则执行）。
- **数值**：数量、金额、日期、比例、单号、物料编码等金标中出现的数字与日期类信息。
- **要素**：结论、状态、对象名称、是否满足条件等金标明确给出的要点。

---

## 二、可选字段与语义等价

标准答案中可附带单据号、仓库编码等便于核对。**若题目未要求逐条列出编码**，被测回答在**语义与事实**上与金标一致即可判对（同 Suite-2 说明）。

---

## 三、各行为类型附加规则

### S1 规范查询

- 金标为唯一依据；数值与要素须与金标一致。

### S2 口语/模糊查询

- 题目可能省略仓库、时间或单据号；金标已按**数据集默认解释**写明（例如默认仓库 WH001、默认快照月 2023-05 等）。
- 判分时：Agent 在合理澄清后给出的结论与金标**事实一致**即视为正确；若 Agent 错误理解指代对象（如认错产品/物料）则判错。

### S3 追问/下钻

- 每题含 **上文假设**；金标假设用户已掌握该上下文。
- 被测回答须与金标在事实上一致；不要求复述上文假设全文。

### S4 异常/边界

- 金标 **预期行为** 列为「不得虚构数据」「应提示未找到」「应说明仅只读」等时，以**是否满足该预期**为主判定。
- 对「数据不存在」类题目：若 Agent 编造单号/数量，**一律判错**。

### S5 多意图混合

- 金标按子意图分条列出（如「意图1…意图2…」）。
- **所有子意图均正确**才判本题正确；任一子意图错误则本题错误。

---

## 四、不合格情形（汇总）

- 数值或要素与金标冲突；遗漏关键要素；结论相反。
- S4 中违反预期行为（尤其是虚构数据）。
- S5 中遗漏任一子意图或子意图错误。

---

## 五、测试方法（建议流程）

1. 将 `ref/suite-3/supplychaindata_1/` 作为问答数据源（或经同步的数仓/BKN）。
2. 按 `questions/` 中 **S1→S5** 顺序执行，题号与 `cases/`、`answers/` 一致。
3. 逐题对照 `answers/` 中金标，按本文件第一至第三节判定。

## 六、测试报告规范

### 6.1 输出位置

```
test_report/suite-3/
```

### 6.2 报告文件

```
test_report/suite-3/
  总体报告.md
  S1_规范查询_测试报告.md
  S2_口语模糊查询_测试报告.md
  S3_追问下钻_测试报告.md
  S4_异常边界_测试报告.md
  S5_多意图混合_测试报告.md
```

### 6.3 分组报告字段

| 内容 | 说明 |
|------|------|
| 题号 | Q01 起 |
| 被测回答 | Agent 实际输出 |
| 标准答案 | 来自 `answers/` |
| 判定结果 | ✅ / ❌ |
| 错误说明 | 否定时填写 |
| 工具调用次数 | 可选 |
| Token 消耗 | 可选 |

### 6.4 总体报告统计

- 总题数 **100**；正确题数；总体准确率；可选工具/Token 汇总；错误题归因与改进建议。
