## 测试用例

> **评分维度**（每题）：
> - **准确性**：数字、状态与基准数据一致
> - **结论优先**：先给判断，再给依据，不以数据罗列开场
> - **行动导向**：回答落到"谁去做什么"的可执行行动
> - **连贯性**：回答末尾有合理的深入引导

## 附录：评分标准详细说明

### 评分维度定义

| 维度 | 满分 | 评分标准 |
|------|------|---------|
| 准确性 | 40分 | 关键数字（数量、日期、单号、状态）与基准数据一致，不得有虚构数据 |
| 结论优先 | 20分 | 第一句话是判断/结论，而非数据；给出明确的是/否/高风险/低风险判断 |
| 行动导向 | 25分 | 回答包含可执行行动（谁、做什么、今天/本周）；不只是数据描述 |
| 连贯性 | 15分 | 回答末尾有合理的追问引导；引导方向与问题情境相关 |

### 合格门槛

- 单题总分 ≥ 70分视为通过
- 准确性 < 25分（关键数字错误）一票否决，直接不通过
- 虚构数字（如凭空捏造供应商名称、PO单号）= 0分

### 快速排查不合格模式

| 不合格模式 | 常见表现 |
|-----------|---------|
| 数据列表开场 | 回答前3行都是数字表格，没有判断 |
| 风险低估 | 把"PO正常"等同于"没有风险" |
| 虚构数据 | 捏造供应商名、单号、到货日期 |
| 无行动 | 回答以"综上所述"或纯描述结尾 |
| 仓库不分 | 把委外加工仓/借用仓视为有效生产库存 |
| MRP误读 | 把"MRP未投放"误读为"有采购缺口"（可能是库存充足） |
